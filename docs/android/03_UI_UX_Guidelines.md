# Flux Android - UI/UX 体验与交互设计规范

## 1. 设计初衷与核心理念

Flux 是一个围绕“个人时间”建立的本地生活管理空间。在 Android 端，UI 设计必须传达以下品牌气质：
**安静、可靠、清晰、克制、有温度。**

- **去压迫感**：不追求强烈的视觉冲击，避免将数据包装成产生压力的“仪表盘”。界面更像是一张整理好的“木质书桌”。
- **先记录，后整理**：降低输入的阻力，尤其是日记编写时不强制要求标题，让用户直接写下想说的内容。
- **允许后悔（可回头）**：删除操作应设计得温和且安全，通过回收站和完善的合并恢复机制，消减操作的恐惧感。

## 2. 视觉规范基础 (Design Tokens)

依托 Jetpack Compose 的 `MaterialTheme` 进行定制，全面支持亮色（Light Mode）与暗色（Dark Mode）主题切换。

### 2.1 色彩语义 (Color Semantics)
避免随意使用系统色，颜色应承载业务逻辑含义：
- **暖黄色 (Warm Yellow)**：代表【日记】。寓意记忆、书写、个人痕迹。用于日记图标、高亮选区。
- **蓝色 (Soft Blue)**：代表【日历事件】。寓意安排、确定性。用于时间胶囊、事件小圆点。
- **柔和红色 (Muted Red)**：代表【待办任务】。寓意行动、优先级。用于高优待办提示。
- **淡橙色 (Pale Orange)**：代表【节假日】。寓意休息、例外、时间节奏。
- **灰色 (Neutral Gray)**：代表【已完成】或【回收站】。通过降低饱和度和透明度，减弱视觉干扰。

### 2.2 字体与排版 (Typography)
- **基准字体**：采用原生系统字体（Roboto/Noto Sans）或针对可读性进行优化的无衬线字体（如 Inter）。
- **Markdown 排版**：对于日记的正文渲染，通过 `AnnotatedString` 或第三方 Markdown 库处理粗体、斜体、列表等，保证字间距与行高（推荐行高 `1.5 - 1.6`）的呼吸感。

## 3. 核心功能路径交互

### 3.1 导航结构
由于移动端屏幕宽度受限，采用 **底部导航栏 (Bottom Navigation)** 承载三个核心页面：
1. **日记 (Diary)**：主屏。列表可无限下拉，上方常驻全局搜索与快捷标签筛选区域。
2. **日历 (Calendar)**：总览。默认不挤满信息，仅展示月视图及节假日标记，需要时由用户点击显示待办/日记的图层。
3. **待办 (Todo)**：行动页。顶部切换“未完成”与“已完成”状态，支持长按拖拽排序。
**设置 (Settings)** 页面入口可放置在顶部工具栏 (Top AppBar) 右侧。

### 3.2 日记编写体验 (The Editor)
- **无标题负担**：直接全屏进入包含 `TextField` 的大区块输入页。
- **沉浸式工具栏**：键盘上方（或页面底部）吸附富文本操作栏，支持快速插入附件（图片、音频、文档选择器）、选择标签 (Tags)、记录天气和心情。
- **日期徽章 (Date Badge)**：在日记列表页，每一张卡片顶部需悬浮清晰的日期徽章（年份/月日/星期），使用户向下滚动时有明确的时间感知。

### 3.3 日历图层与“安全感” (Calendar Layers)
- **日历格子标记**：使用不同颜色的小圆点或极简色块（蓝=事件，红=待办，黄=日记）进行标记。为了可访问性，可以在长按/日详情弹窗中配合文字明确说明。
- **图层开关**：日历页顶部提供可横向滑动的 `Filter Chips`，用于独立开关“显示待办”、“显示事件”、“显示日记”等图层，满足“不想看任务”时的减压需求。

### 3.4 待办与拖拽 (Todos & Drag-Drop)
- **完成置底**：当用户点击勾选框 (Checkbox) 完成一个任务时，该任务应以流畅的动画变为灰色，并自动滑动至列表最底部（或移动到“已完成”收纳区域），不突兀地消失。
- **长按拖拽**：在列表左侧提供拖拽手柄 (Drag Handle) 或通过长按卡片触发重新排序交互，使用户能随心掌控优先级。

## 4. 关键状态设计 (State Design)

### 4.1 空状态 (Empty States)
必须采用中性、友善的文案，禁止指责用户。
- 推荐：“还没有日记，写点什么吧。”
- 避免：“你的日记为空！赶紧记录！”
- 视觉表现：搭配一个柔和、低饱和度的插画或极简 Icon。

### 4.2 删除与恢复状态 (Delete & Restore)
- **软删除操作**：左滑菜单触发删除（配合红色的删除图标），执行后界面给出 `Snackbar`，内含明显的【撤销 (Undo)】按钮。
- **回收站列表**：此列表内的项必须有别于正常内容，使用斜线背景、降低透明度或“回收站”专属角标，防止用户误将删除内容视为正常待办。

### 4.3 表单按钮与反馈
- 核心的正向操作（如“保存”、“合并恢复”）使用主题品牌色填充。
- 破坏性操作（如“彻底清除附件”）要求使用**红色边框+文字（OutlinedButton）**，并强制弹出二次确认弹窗 (Dialog)。
- 明确温和的 Toast 提示（如“日记已合并到当天原日记”）。
